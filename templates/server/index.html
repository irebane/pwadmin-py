Game Link Instances

Game Link Daemon instances

Saving any value fully regenerates the required sections in gamesys.conf and gmserver.conf — no manual file editing needed.

Each instance N listens for clients on port 2900(N−1) and for game zones on port 2930N. Each handles up to 3,000 concurrent connections.

⚠ Server restart required for changes to take effect.

DB Workers

Game Database Daemon workers

Each worker is a forked process that holds one MySQL connection and handles character saves, item operations, and trades. Uses ~9.5 MB RAM per worker.

Rule of thumb: 1 worker per 50 concurrent players, minimum 2. For a small family server 2 is plenty.

Server Status shows total processes, not just workers. 6 fixed overhead processes are always present (parent, I/O, timer, dispatcher forks), so total = setting + 6. Setting 2 → 8 shown in status.

⚠ Server restart required for changes to take effect.

Name Workers

Unique Name Daemon workers

Consulted only during character creation and rename to enforce name uniqueness. Not involved in normal gameplay at all.

1 worker is sufficient for any server size — name checks are infrequent and fast.

Server Status shows total processes, not just workers. 6 fixed overhead processes are always present, so total = setting + 6. Setting 1 → 7 shown in status.

⚠ Server restart required for changes to take effect.
